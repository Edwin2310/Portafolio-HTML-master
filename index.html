    Helpdesk (Soporte Técnico)

  • Este proyecto es un sistema de Helpdesk para gestionar las solicitudes (tickets) de los usuarios de forma ordenada y eficiente: los usuarios crean tickets, el administrador los asigna al personal disponible y todo queda registrado para evitar acumulación de trabajo.

    1. Nuevo Ticket: formulario para registrar un incidente con selección de tipo de problema, descripción y generación de ID único.
    2. Comprobar Estado: consulta introduciendo correo e ID para ver el estado actual del ticket.
    3. Calendario: control de fechas y tiempos según las asignaciones.
    4. Administrar Tickets: vista de todos los tickets filtrados por estado (Pendientes, En Proceso, Resueltos, Rechazados).
    5. Consultar Ticket: interfaz para responder y cambiar el estado de un ticket.
    6. Dictamen: sección para que el administrador suba archivos en formato Word.
    7. Informe Técnico: creación de un dictamen cuando un dispositivo está dañado, con opción de insertar hasta 3 imágenes como anexos.
    8. Entrada/Salida de Equipo: registro del trabajo de los técnicos y obtención de estadísticas de las fallas más frecuentes.
    9. Bitácora: formulario en el que los técnicos documentan soluciones para referencia de otros.
    10. Gestión de Usuarios: alta de usuarios y administradores mediante código de empleado.
    11. Panel de Usuarios: editar datos, cambiar estado, restablecer contraseña y eliminar usuarios.
    12. Notificaciones por Correo: envío automático de emails al usuario y al administrador de Helpdesk.

    Proyecto Nuevo SIGEM (Sistema de Tickets para el SNE-911)

  • Sistema de gestión de emergencias 911: el ciudadano llama, el operador crea el ticket, el despacho coordina la ayuda y el supervisor lo cierra.

    1. Login por rol: acceso para Administrador, Supervisor y Despacho.
    2. Dashboard: métricas y gráficos de tickets (totales, abiertos, cerrados).
    3. Gestión de tickets: creación, consulta y filtrado de incidencias.
    4. Detalle de ticket: visualización de datos específicos y chat con despacho o supervisor.
    5. Notificaciones: vista de tickets asignados a cada despacho.
    6. Perfil y sesión: cambio de contraseña y cierre de sesión.

    SIGAES (Sistema de Gestión Académica en la Educación Superior)

  • Sistema integral para soportar procesos administrativos, seguridad, reportería, bitácora, mantenimiento, gestión y consulta de solicitudes, y análisis curricular en la Educación Superior departamento de la UNAH.

    1. Procesos Generales: gestión de solicitudes, admisión, actas y agendas.
    2. Seguridad: autenticación, control de acceso, gestión de sesiones y respaldo de datos.
    3. Reportería: generación y exportación de informes configurables (PDF, Excel, CSV).
    4. Bitácora: registro de eventos, seguimiento de modificaciones y control de logs.
    5. Mantenimiento: administración de catálogos, usuarios, roles y permisos.
    6. Solicitudes: registro, seguimiento, revisión documental y cierre de trámites.
    7. Consulta de Solicitudes: búsqueda avanzada, historial, exportación de consultas y estadísticas.avanzada.
    8. Análisis Curricular: revisión de planes de estudio, dictamen académico y generación de resoluciones.
